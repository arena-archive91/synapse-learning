# Internationalization (i18n)

Synapse supports **English** and **Greek** at the UI shell level. Lesson and note content follow the uploaded material language; UI chrome is partially translated.

## Configuration

**Settings → Language** → `UserSettings.language`: `'en' | 'el'`

Stored in `synapse:session-v2` with other preferences.

## Architecture

| Module | Role |
| ------ | ---- |
| `src/lib/i18n.ts` | Translation keys, `t(key, lang)`, `I18nContext` |
| `App.tsx` | Provides `I18nContext` to all views |
| `Shell.tsx` | Sidebar labels via `t('dashboard')`, etc. |
| `settingsEffects.ts` | `t()` helper for pedagogy strings |

## Hardcoded English (known)

- `mockData.ts` — demo only, hidden by default
- `domainContent.ts` — legacy Cournot/marketing; avoid in production path
- Analytics chart series names
- Some Agent offline templates
- Server API error strings

## Adding translations

1. Add key to `I18nKey` union in `i18n.ts`
2. Add `en` and `el` strings in the translations map
3. Replace hardcoded string with `t('yourKey')` or `useI18n().t`

## Content language vs UI language

Uploaded notes may be Greek while UI is English (or vice versa). Extractors and BM25 handle both scripts. Lesson summaries use `lang` for UI framing only; excerpts stay in source language.
